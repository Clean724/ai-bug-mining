# LLM 交互记录填写模板

每次真实交互后，将内容追加到 `work/llm_chat_log.json`。

```json
{
  "turn": 1,
  "role": "user",
  "content": "请只基于下面提供的源码片段进行安全与稳定性审计，不要依赖项目名称、版本号、CVE、issue、PR 或任何已知漏洞信息。..."
}
```

```json
{
  "turn": 1,
  "role": "assistant",
  "content": "漏洞是否成立：可疑但需验证..."
}
```

## 注意事项

- 不要在用户提示词中写项目名称。
- 不要写指定版本号。
- 不要写“这个版本有某某 bug”。
- 不要引用 CVE、issue、PR 或历史修复。
- 可以提供源码片段、文件路径、行号和本地运行结果。
- 可以要求模型生成测试、复现脚本、修复建议。
